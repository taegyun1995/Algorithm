|:---:|:---|
| 파일 | P16173_v4.java |
| 날짜 | 2026-01-27 |
| 알고리즘 | `DFS (스택 기반)` |
| 자료구조 | `Stack`, `2차원 배열` |
| 시간복잡도 | O(N²) |
| 공간복잡도 | O(N²) |

**잘한 점**
- 스택 기반 DFS 완벽 구현
- 방문 체크 올바름
- 경계 조건 올바름
- 이동 방향 (오른쪽/아래) 분리 완료
- 도착점(-1) 체크 올바름

**개선할 점 (정확성)**

없음 - 정답!

**최적화 제안**

1. **Stack → ArrayDeque**
   - 현재: `Stack<Jelly>` (Vector 기반, 동기화 오버헤드)
   - 개선: `Deque<Jelly> stack = new ArrayDeque<>();`
   - 효과: 약 2배 빠름

2. **방향 배열로 단순화**
   - 현재: if문으로 방향 분기
   - 개선:
     ```java
     int[] dr = {0, 1};
     int[] dc = {1, 0};
     next_row = now_row + dr[i] * move;
     next_col = now_col + dc[i] * move;
     ```
   - 효과: 코드 간결화, 확장성 향상

3. **조건문 개선**
   - 현재: `if (i == 0)` ... `if (i == 1)`
   - 개선: `if (i == 0)` ... `else`
   - 효과: 불필요한 비교 제거

4. **객체 생성 줄이기**
   - 현재: `new Jelly(row, col)` 매번 생성
   - 개선: `int[]` 배열 사용 또는 `ArrayDeque<int[]>`
   - 효과: 힙 할당 감소
