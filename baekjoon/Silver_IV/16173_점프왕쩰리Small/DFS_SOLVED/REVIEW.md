# 백준 16173번: 점프왕 쩰리 (Small) 리뷰

| 구분 | 내용 |
|:---:|:---|
| 파일 | P16173.java |
| 날짜 | 2026-01-27 |
| 알고리즘 | `DFS` |
| 자료구조 | `Stack`, `2차원 배열` |
| 시간복잡도 | - (무한루프 발생) |
| 공간복잡도 | O(N²) |

**잘한 점**
- BufferedReader, StringTokenizer 사용으로 입력 처리 효율적
- Point 클래스로 좌표 캡슐화
- 스택 기반 DFS 시도

**개선할 점**

1. **시작점이 잘못됨**
   - 왜? 문제에서 시작점은 (0, 0)인데, 현재 코드는 모든 칸에서 DFS 시작
   - `dfs(0, 0)`만 호출해야 함

2. **이동 방향 오류**
   - 왜? 오른쪽과 아래 **따로** 이동해야 함
   - 현재: `next_row = now_row + move`, `next_col = now_col + move` (대각선 이동)
   - 올바른 방식: 오른쪽 `(row, col + move)`, 아래 `(row + move, col)` 각각 탐색

3. **방문 체크 없음 → 무한 루프**
   - 왜? visited 배열 없이 같은 칸을 반복 방문할 수 있음

4. **스택 + 재귀 혼용 (비효율)**
   - 왜? 36번 줄에서 스택 사용하면서 51번 줄에서 재귀 호출
   - 둘 중 하나만 사용해야 함

5. **경계 조건 오류**
   - 왜? 44번 줄 `next_row > N && next_col > N`은 둘 다 초과해야 continue
   - `||`(OR)로 변경해야 하나라도 범위 밖이면 스킵

6. **도착 조건 오류**
   - 왜? 도착점은 MAP 값이 -1인 칸, 또는 (N-1, N-1) 위치
   - 현재 next_row, next_col로 체크하는데, 이동 전 현재 위치로 확인해야 함

7. **"Hing" 출력 누락**
   - 왜? 도달 못 할 경우 "Hing" 출력이 없음

8. **결과 출력 후 종료 안 함**
   - 왜? "HaruHaru" 출력 후 프로그램이 종료되지 않음
