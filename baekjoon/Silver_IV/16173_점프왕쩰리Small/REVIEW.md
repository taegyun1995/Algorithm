# 백준 16173번: 점프왕 쩰리 (Small) 리뷰

---

## v1 (2026-01-23) - P16173.java

| 구분 | 내용 |
|:---:|:---|
| 알고리즘 | `DFS (스택 기반)` |
| 자료구조 | `2차원 배열`, `Stack` |
| 시간복잡도 | - (무한 루프 발생) |
| 공간복잡도 | O(N²) |

**잘한 점**
- BufferedReader, StringTokenizer 사용으로 입력 처리 효율적
- Point 클래스로 좌표 캡슐화
- 스택 기반 DFS 시도

**개선할 점**

1. **시작점이 잘못됨**
   - 왜? 문제에서 시작점은 (0, 0)인데, 현재 코드는 모든 칸에서 DFS 시작
   - `dfs(0, 0)`만 호출해야 함

2. **이동 방향 오류**
   - 왜? 오른쪽과 아래 두 방향으로 따로 이동해야 함
   - 현재: `next_row = now_row + move`, `next_col = now_col + move` (대각선 이동)
   - 올바른 방식: 오른쪽 `(row, col + move)`, 아래 `(row + move, col)` 각각 탐색

3. **방문 체크 없음 → 무한 루프**
   - 왜? visited 배열 없이 같은 칸을 반복 방문할 수 있음
   - 특히 0인 칸에서 무한 루프 발생

4. **스택 + 재귀 혼용 (비효율)**
   - 왜? 32번 줄에서 스택 사용하면서 51번 줄에서 재귀 호출
   - 둘 중 하나만 사용해야 함

5. **경계 조건 오류**
   - 왜? 44번 줄 `next_row > N && next_col > N`은 둘 다 초과해야 continue
   - `||`(OR)로 변경해야 하나라도 범위 밖이면 스킵

6. **도착 조건 오류**
   - 왜? 47번 줄에서 `next_row`, `next_col` 체크하는데, 이동 전 현재 위치가 도착점인지 확인해야 함
   - 또한 도착점은 MAP 값이 -1인 칸

7. **"Hing" 출력 누락**
   - 왜? 도달 못 할 경우 "Hing" 출력이 없음

8. **결과 출력 후 종료 안 함**
   - 왜? "HaruHaru" 출력 후 `return` 또는 `System.exit(0)`으로 종료해야 함

**올바른 접근법**
```
1. (0, 0)에서 DFS 시작
2. 현재 칸이 -1이면 "HaruHaru" 출력 후 종료
3. 현재 칸의 값만큼 오른쪽으로 이동 시도
4. 현재 칸의 값만큼 아래로 이동 시도
5. 방문 체크로 중복 방문 방지
6. DFS 완료 후 도착 못 했으면 "Hing" 출력
```
