## v1 (2025-01-03) - P1000.java

| 구분 | 내용 |
|:---:|:---|
| 알고리즘 | `단순 연산` |
| 자료구조 | `int` |
| 시간복잡도 | O(1) |
| 공간복잡도 | O(1) |

**잘한 점**
- BufferedReader + StringTokenizer 조합으로 빠른 입력 처리
- 간결하고 명확한 코드

**개선할 점**

1. **static 변수 → 지역 변수로 변경**
   - 왜? `static` 변수는 클래스 레벨에서 공유되어 메모리에 계속 남아있음
   - 이 문제에서는 `main()` 안에서만 쓰고 끝나므로 지역 변수가 적합
   - 지역 변수는 메서드 종료 시 자동으로 메모리 해제됨
   - 습관적으로 `static`을 쓰면 나중에 멀티스레드 환경에서 버그 원인이 될 수 있음

2. **변수명 N, M → A, B**
   - 왜? 문제에서 "A와 B를 입력받아"라고 명시함
   - 코드 읽는 사람이 문제와 코드를 매칭하기 쉬워짐
   - N, M은 보통 "개수"나 "크기"를 나타낼 때 관례적으로 사용

**배운 점**
- Java에서 Scanner보다 BufferedReader가 빠름
- 간단한 문제라도 변수명은 문제 설명과 일치시키기
