**잘한 점**
- 문제 조건을 주석으로 깔끔하게 정리
- Stack 기반 반복 DFS로 구현해 재귀 깊이 걱정 없음
- 방향 배열(DR, DC)로 오른쪽/아래 이동을 일관되게 처리
- visited 배열로 중복 방문 방지
- 범위 체크 `>= N`으로 정확하게 처리

**개선할 점 (정확성)**

없음 — 정답

**최적화 제안** (정확성 통과 후)

1. **Stack → ArrayDeque**
   - 현재: `Stack<Jelly>` (synchronized)
   - 개선: `ArrayDeque<Jelly>`
   - 효과: 동기화 오버헤드 제거 (N ≤ 3이라 실질 차이는 미미)

2. **Jelly 객체 → int 인코딩**
   - 현재: 좌표마다 Jelly 객체 생성
   - 개선: `row * N + col` 하나의 int로 인코딩
   - 효과: 객체 생성/GC 비용 절감 (N ≤ 3이라 실질 차이는 미미)
