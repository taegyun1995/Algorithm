# 백준 1303번: 전쟁 - 전투 리뷰

| 구분 | 내용 |
|:---:|:---|
| 파일 | P1303.java |
| 날짜 | 2026-01-24 |
| 알고리즘 | `DFS (스택 기반)` |
| 자료구조 | `2차원 배열`, `Stack` |
| 시간복잡도 | O(N×M) |
| 공간복잡도 | O(N×M) |

**잘한 점**
- DFS 스택 기반 구현 시도
- 방문 체크 배열 사용
- 4방향 탐색 dx/dy 배열 활용
- push 시점에 방문 체크 (중복 push 방지)

**개선할 점**

1. **입력 파싱 오류 (28번 줄)**
   - 왜? `s.charAt(i)` → `s.charAt(j)`여야 함
   - 현재 코드는 열(j)을 순회하면서 행 인덱스(i)로 문자를 가져옴
   - 결과: 잘못된 보드 데이터 저장

2. **power 초기값 중복 (42, 47번 줄)**
   - 왜? `b_power = 1`로 초기화 후 dfs에서 시작 노드도 `power++` 함
   - 결과: 그룹 크기가 실제보다 1 큼
   - 해결: 초기값을 0으로 하거나, dfs에서 시작 노드 카운트 방식 수정

3. **else if 미사용 (46번 줄)**
   - 왜? `if ('B' == target)` 후 `if ('W' == target)` 사용
   - B와 W는 상호 배타적이므로 `else if`가 적절
   - 현재도 동작하지만 불필요한 조건 검사

4. **target 파라미터 불필요**
   - 왜? dfs 호출 시 이미 `BOARD[row][col]`로 target 알 수 있음
   - dfs 내부에서 `BOARD[row][col]`로 직접 접근 가능
